Creating a Standalone Section

"The HTML section element represents a standalone section — which doesn't have a more specific semantic element to represent it — contained within an HTML document. Typically, but not always, sections have a heading."

Semantic Considerations

The section element is flexible, and can be used to start any kind of new information. Usually the section element is used to section off any new information in the HTML, whether this be a new section following the same format, or a new section in a different format. Headings are also normally used at the beginning of sections to specify the start of a new section.

Common Attributes and Values

The section element only includes the global attributes.

Accessibility Considerations

The section element has the Implicit ARIA role of region, but only if the element has an accessible name. Otherwise, there would be no corresponding role.

The section element has the following Permitted ARIA roles:

  • alert
  • alertdialog
  • application
  • banner

Example Uses

Creating two different sections within the main element, having them describe different ideas. 

<section>
        <h4>The First Task</h4>
        <p>The first task that must be completed</p>
</section>
<section>
        <h4>The Second Task</h4>
        <p>The second task that must be completed</p>
</section>

Creating two different sections within the footer element, having them contain different links. 

            
<section>
            <h4>Social Media Links</h4>
            <p>Linked Link</p>
</section>
<section>
            <h4>Resources Links</h4>
            <p>Mozilla Developer Network Link</p>
</section>
