MDN Definition of <head>:

The HTML <head> element contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets.

Semantic Considerations

The <head> element is typically used at the beginning of a file, within the <html> element. Usually, the <head> element contains the <title> element, which is used to give a name to the tab of the webpage (look at this page's title on its tab!). The <head> element also typically includes the <meta> element, which wraps around metadata for the file. Metadata is not seen on the webpage itself, but it may define the character set, scale the webpage to a certain size, or provide information about the page such as the name(s) of the author(s).

Accessibility Considerations

  • Meta data will not be displayed to reader, it is just to provide information about the webpage
  • <title> element is necessary to provide a purpose for the page.

Example Uses

Head is used after the opening html tag to wrap around meta data and provide a title for the webpage. It has 2 elements, title and meta. The title is used to provide a purpose for the page and also displays the title of the webpage on the browser window and the tab. The meta tag is used to provide information about the webpage which is not visible to the reader.

<!doctype html>
<html>
<head> 

   <title>Documents Title</title>
   <meta name="description" content="A review page for the block element 'head.'" />
   <meta name="author" content="Chaitra Katragadda, Tyler Pozarycki" />
</head>
</html>
