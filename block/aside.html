Aside elements are used for content that goes to the side

The HTML [aside] element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as side bars or call-out boxes.

Semantic Considerations

This element is a "content sectioning" element, so you should use this element to wrap around any elements you'd like in that section, much like section and article elements, except aside elements should be used to mark content that is indirectly related to the main content. Aside elements typically wrap written content, such as headings, paragraphs, and lists or embedded content, and they can be wrapped within body or footer elements as well as inside section, article, and even paragraph elements.

Common Attributes and Values

  • aria-label attribute if there are multiple aside/ complementary elements on the page to help assistive technologies
  • Any global attributes
  • Any event attributes

Accessibility Considerations

Using the aside element should automatically communicate to assistive technology that the content is separated from the main content on the page, which takes care of the ARIA role=complementary, but you may still want to include this role.

If there's more than one complementary or aside element on the page, you should provide a descriptive label for each landmark using the aria-label attribute: aria-label=""; so that assistive technology can identify the purpose of each landmark element.

Other permitted ARIA roles include:

  • feed
  • none
  • note
  • presentation
  • region
  • search
