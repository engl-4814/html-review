Listing Things

The HTML <li> element is used to represent an item in a list. It must be contained in a parent element: an ordered list (<ol>), an unordered list (<ul>), or a menu (<menu>). In menus and unordered lists, list items are usually displayed using bullet points. In ordered lists, they are usually displayed with an ascending counter on the left, such as a number or letter.

Semantic Considerations

The <li> element is expected to show list items within a list. These items can be numbered, bulleted, or be displayed as menu items.

Content that is usually wrapped by the <li> element are items in a list.

Common Attributes and Values

  • The "value" attribute can be attached to <ol> to designate what number the list starts on

Accessibility Considerations

  • Many lists are directories to other pages or various places in the same page and there is a role called "button" which denotes a link to that can be activated.
  • The "checkbox" role could be used with lists especially in surveys.
  • A "listbox" role may denote a large number of options in a survey as opposed to individual "checkbox" roles
  • Many times a list is a menu on a page that allows people to go places on the page or other pages that can be denoted with the "menu" role
  • A "form" role may be good for denoting a section of a website

Example Uses

Create a Navigation Bar 

A list, ordered or unordered, is a good way to create a navigation bar on a website.
            Navigation lists are connected to links that direct to other sections in the page.

External Links and Social Media 

Using unordered lists for this is a good way to create a defined section for these.
            These lists are connected with links and maybe buttons that connect to other pages
